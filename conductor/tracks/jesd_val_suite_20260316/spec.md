# Track Specification: Implement JESD204 configuration validation suite

## 1. Goal
The objective of this track is to implement a comprehensive validation suite for JESD204 configurations generated by `pyadi-jif`. This suite will ensure that parameters for converters, clocks, and FPGA links are within valid ranges and consistent with each other, reducing the risk of invalid hardware settings.

## 2. Requirements
- **Generic JESD204 Parameter Validation:** Validate L, M, F, S, K, and other standard JESD204 parameters.
- **Component-Specific Rules:** Implement validation logic specific to ADI chips (e.g., AD9081/AD9084 lane rate limits, HMC7044 output frequency ranges).
- **System-Level Consistency:** Verify that clocks and converter sample rates are compatible with the JESD204 link rates.
- **Reporting:** Generate clear reports of valid and invalid parameters, highlighting specific rule violations.
- **Integration:** Expose the validation suite through the JIF Tools Explorer (Streamlit) for interactive use.

## 3. Core Components
- **`ValidationEngine`:** Base class for running validation rules.
- **`JESD204Rules`:** Standard rules for the JESD204 protocol.
- **`ConverterRules`:** Device-specific validation for ADI ADCs and DACs.
- **`ClockRules`:** Device-specific validation for ADI clock generators and buffers.
- **`SystemValidator`:** High-level validator for complete end-to-end signal chains.

## 4. Acceptance Criteria
- [ ] A base validation engine exists that can run multiple rules against a configuration.
- [ ] Generic JESD204 parameter checks are implemented and passing for standard modes.
- [ ] Validation rules for at least one major converter (e.g., AD9081) and one clock (e.g., HMC7044) are implemented.
- [ ] System-level compatibility checks (e.g., lane rate vs. sample rate) are implemented.
- [ ] Validation status and errors are visible in the JIF Tools Explorer.
- [ ] Test coverage for the new validation logic is >90%.
